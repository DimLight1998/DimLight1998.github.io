Rust 学习笔记

发表于 2018-11-10 | 更新于 2019-03-11

Rust 学习笔记

基础

工具链

Rust 使用 cargo 进行项目管理。

  • 使用 cargo 建立一个新项目：cargo new project_name --bin。
  • 使用 cargo 建立一个新模块：cargo new module_name --lib。
  • 编译并运行： cargo run。
  • 只编译：cargo build。
  • 只检查是否能够通过编译：cargo check。
  • 更新依赖：cargo update。
  • 运行测试：cargo test。

变量与可变性

  • Rust 使用 let 声明变量，建议使用 snake_case 命名。
  • Rust 使用 const 声明常量，建议使用 SCREAMING_SNAKE_CASE 命名。
  • Rust 变量默认不可变，如果需要可变的话，需要加上 mut。

Rust 具有 Shadowing 的特性，可以将声明一个与同作用域已经声明的变量名称相同的变量名。

数据类型

标量类型

  • 整数：i8, i16, i32, i64, isize, u8, u16, u32, u64, usize
  • 浮点数：f32, f64
  • 布尔值：bool
  • 字符：char，使用 Unicode

整数字面量中可以出现下划线 _ 方便阅读，使用 0x, 0o, 0b 来切换进制。

复合类型

Rust 有元组和数组两种复合类型。

  • 元组索引使用 .，例如 tup 是一个元组，取出第一个元素为 tup.0。
  • 元组支持解构 let tup: (i32, bool) = (123, true); let (x, y) = tup;。
  • 数组是定长的，编译时确定长度，数组索引使用中括号。数组的类型标注示例 let a: [i32; 5] = [1, 2, 3, 4, 5];。

函数

先区分 statement 和 expression。

注意 Rust 中的 expression 可以是大括号括起的若干 statement 加上一个 expression。可以写出这样的代码：let y = {let x = 3; x + 1};。

Rust 中函数定义使用 fn，需要为参数和返回值使用类型标注。函数命名建议 snake_case。

函数是一连串的 statement，外加最后的一个 expression，这个 expression 将会作为返回值返回。注意不要在最后一个 expression 后加上分号，否则会变成 statement。如果要提前返回，使用 return。最后一个 expression 使用 return 是不好的风格。

注释

Rust 使用 // 作为单行注释，使用 /// 对文档进行注释，使用 //! 对模块进行注释。后面两种注释内部使用 Markdown。

控制流

if expression

Rust 中的 if 是 expression，这说明你可以写出这样的代码：let x = if condition_1 {1} else if condition_2 {2} else {3};。

当然像其它语言一样控制流程也是可以的，注意 if 后面可以省掉括号。

loop 无限循环

Rust 使用 loop 进行无限循环，不能带任何条件。

while 条件循环

Rust 使用 while 进行条件循环。和 if 一样不用带括号。

for 容器遍历

Rust 使用 for ... in ... 对容器进行遍历，并且只有这一种 for 循环语法。

所有权系统

所有权（系统）是 Rust 最独特的功能，其令 Rust 无需垃圾回收（garbage collector）即可保障内存安全。因此，理解 Rust 中所有权如何工作是十分重要的。

所有权

Rust 不使用垃圾回收，也不要求程序员手动进行内存的分配和回收，而是通过所有权的概念管理内存。

Rust 中的每个（堆上的）值（例如 String 类型中实际存放字符串的内存）都有一个 所有者，并且 只可能有一个所有者。因为所有者总是在栈上，当所有者离开作用域的时候，值被销毁（例子中是内存被释放，由于只可能有一个所有者，这是安全的）。

此外 Rust 中有一个名叫 Copy 的 trait，实现了它的类型的行为与上述不同。如果一个类型是 Copy 的，那么它十有八九都是在栈上分配，和堆没有关系，这个时候就没有内存管理的问题了。实现了 Copy trait 的类型有所有的标量类型以及所有分量都是 Copy 的元组。

为了保证每个堆上的值只有一个所有者，所有权会在不同的变量之间进行转移。拥有所有权的变量在所有权被转移后无法继续拥有所有权。例如：

拷贝——所有权不转移

如果希望拷贝堆上的数据可以使用相应类型的拷贝方法。这种情况下所有权不会转移，而是堆上有两份相同的数据，它们拥有不同但是唯一的所有者。

与函数和返回值有关的情况

所有权的转移也可能发生在函数调用时（将所有权转移给了参数）和函数返回时（注意返回值总是以表达式存在的）。这说明一旦调用了一个函数并闯入了参数，如果这个函数没有返回参数，那么这个参数在函数调用结束的时候对应的堆内存就被回收了，函数调用完成后后面无法再使用之前传入的那个参数了！

引用和借用

引用 和 借用 实际上说的是同一个东西！所以后面只会提到 引用。

因为转移所有权看起来就很不方便（如果要写一个计算字符串长度的函数，将字符串传入函数后想重新得到所有权就要让函数返回来，同时要得到长度的话还要一个额外的返回值），所以有了引用的概念。

一个类型和它的引用类型是两种不同的类型，类型本身在离开作用域的时候会释放对应区域的内存，但是引用离开作用于的时候什么都不会发生。对下面的函数：

引用默认是不可变的，如果需要有可变的引用，要加上 mut 关键字。需要注意的是一个作用域中如果有一个可变引用，就不能再有其它的任何引用。这是 Rust 为了在编译时消除数据竞争问题做的设计。

OK，不过为啥有 String 和 str 的区别呢？我的理解是 &str 是对内存中那一块连续区域的引用，而 &String 是对字符串这个整体的引用，前者似乎更加底层。另外说一句，使用 let a = "hello world"; 的话 a 就直接是 &str 了，这个字符串是直接在数据区的。这个字符串是无法更改的，即时使用了 let mut a = "hello world"; 也只是让 a 可以成为其它 str 的引用。如果引用了一个字符串，这个字符串的内容可变和自己还可以引用别的东西是有区别的！

数组也可以对 slice 进行引用，类型形如 &[i32]。

元组结构体

元组结构体实际上就是对元组取名字以让它们具有不同的类型，元组结构体的字段没有名字。元组结构体也可以进行解构。

类单元结构体

类单元结构体没有任何字段。

给结构体添加方法

结构体可以拥有自己的方法，并且也可以拥有所谓的 关联函数。这个概念类似于 OOP 中的实例方法和静态方法。

结构体的方法都要放在一个 impl 块中，如果这个方法第一个参数是 &self 或者是 &mut self，则表明这个是实例的方法，否则它是一个静态的方法。一个结构体可以对应多个 impl 块。

至于为什么要这么设计，我估计是方便对已有的结构体的方法进行扩展（类似 C# 中的扩展方法）。

模块系统

Rust 拥有一个好用的模块系统，并且有一个伴随着的测试系统。

模块系统的定义

使用 mod 关键字定义模块，模块可以嵌套定义。模块和 crate 都推荐使用 snake_case 命名。

也可以只声明一个模块，而将定义放到其它的文件中。当你要这么干的时候，需要将整个模块按照某种特定的在文件系统中的格式组织起来。

使用 pub 关键字将模块中的内容标记为对外可见的。

引用模块

使用 extern crate create_name; 来引入外来的模块，使用 use 来将一个模块中的内容引入当前命名空间。

use 可以跟：

  • use a::series::of::nested_modules;
  • use TrafficLight::{Red, Yellow};
  • use TrafficLight::*;

可以在 use 中使用 super 来访问上一级模块。

通用集合类型

Rust 提供了一些常见的集合类型，例如 Vec、String 和 HashMap。

错误处理

Rust 的错误可以分成两种：可恢复的错误和不可恢复的错误。

对于不可恢复的错误，使用 panic! 宏会使程序结束。如果程序带有调试信息，还会打印调用栈。

对于可恢复的错误，一般是使用 Result 将其包装起来，Result 的定义是：

一般使用模式匹配处理。每个 Result 都有一个 unwrap 方法，这个方法可以在 Result 正常的时候返回值，错误的时候调用 panic!；也有一个 expect 方法，这个方法与 unwrap 类似，不过可以指定 panic! 的内容。

错误可以继续传播上去，让更高层的函数进行处理。这样的函数返回类型一般是 Result。在这样的函数中可以使用 ? 操作符来简化样板代码。当在这样的一个函数中调用一个返回值为 Result 的函数时，如果调用最后加上了 ?，那么在成功时会拿到成功的值，失败时会直接返回包装于 Result 中的错误，并且是直接将得到的错误返回。

测试

将一个函数标记为需要测试的，在函数签名上方加上 #[test] 标记。

将一个模块标记为需要测试的，在模块上方加上 #[cfg(test)] 标记。这个标记将会告诉编译系统只有在需要测试的时候才进行编译。

在测试的函数中，使用 assert!、assert_eq! 和 assert_ne! 等宏来进行检查；在函数上添加 #[should_panic] 或更详细的 #[should_panic(expected = "infomation_here")] 来标记这个函数运行中应当 panic。

匿名函数可能实现了 Fn、FnOnce 和 FnMut 这三个 trait 中的若干个，区别在于 Fn 获取环境中不可变引用；FnMut 获取环境中可变引用；FnOnce 获取环境中变量的所有权。

迭代器

一般使用 iter 方法获得一个容器的不可变引用迭代器，使用 iter_mut 获得可变引用迭代器，使用 into_iter 获得返回所有权的迭代器。

迭代器都实现了 Iterator trait，实现了这个 trait 的类型都具有 next 方法可以得到下一个值，包装在 Option 中。当遍历完成时，返回 None。如果要实现自己的 Iterator，只用实现 Iterator trait 以及 next 方法。

迭代器的方法一般分成两种，一种是类似 sum 这样的执行一次就会消耗掉迭代器的，另一种是 map 这样的会产生一个新的迭代器的（Rust 中迭代器是惰性的，没有使用前不会进行求值）。

智能指针

Box<T>

使用 Box<T> 可以将一个标量类型放在堆上：let a = Box::new(3);，操作后堆上有 3，同时栈上有一个指向 3 的指针。

使用 Box<T> 的目的可能是定义递归结构。如果递归结构在栈上分配，编译器不知道这个结构的大小，是不可能做到的。

Deref trait

Deref trait 要求实现 deref 方法，实现了这个 trait 后，一个智能指针可以像普通引用那样被使用。如果智能指针为 p，那么 *p 实际上执行的是 *(p.deref())。

Drop trait

实现了 Drop trait 的变量在离开作用域的时候会做一些事情，具体的是这个 trait 要求实现的 drop 方法。这个方法不能手动调用，不过，可以使用 std::mem::drop 函数来手动清理一个智能指针。

Rc<T>

Rc<T> 是引用计数的指针。

  • 使用 Rc::new 创建第一个引用。
  • 使用 Rc::clone 对其它引用进行克隆。
  • 使用 Rc::strong_count 获得当前强引用计数。
