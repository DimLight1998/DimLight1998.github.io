发表于 2018-11-23 | 更新于 2019-02-28

代码优化-1

代码优化就是使程序运行更快、效率更高的对程序的等价变换。

流图

基本块

基本块是一段连续的三地址指令序列，满足 控制流只能从序列首进入，只能从序列尾退出，序列中不会出现跳转和停机。

将一段程序划分为基本块的算法需要先确定 首指令，即作为某个基本块的第一条指令的指令。首指令可能是指令序列的第一条指令、跳转指令本身和紧接着跳转指令后的指令。首指令之间的指令片段即为一个基本块。

流图

流图是一张有向图，反映了控制流的转移。其结点对应着基本块，两个结点之间存在一条边当且仅当这两个结点对应的基本块存在可能的控制流转移。结点 M 指向结点 N 有可能是 M 在某种情况下会跳转向 N，也可能是 M 执行完了后不会跳转，但是会顺着进入 N。

常用的代码优化方法（一）

优化分类

按照优化的目标可以分为针对中间表示的 机器无关优化 和针对目标代码的 机器相关优化。

按照优化的范围可以分为针对单个基本块的 局部代码优化 和针对多个基本块的 全局代码优化。

删除公共子表达式

如果一个表达式中的子表达式之前计算过，下次遇到时表达式中的元素的值未改变，则称为公共子表达式，可以删去。

如果子表达式的上次计算是在同一个基本块中完成的，称为 局部公共子表达式，否则应该是在不同基本块中完成的，称为 全局公共子表达式。

一个重要的问题是如何识别公共子表达式。

常用的代码优化方法（二）

删除无用代码

有些代码是无用的，例如执行了公共子表达式删除后会引入一些复制语句如 x = y，复制传播就是尽可能使用 y 代替 x。

使用了复制传播后又有可能产生死代码，这种代码的执行结果永远不会被用到，可以直接删除。

常量合并

在编译时就识别出常量的表达式，并计算出结果替代这些表达式。

代码移动

对于循环，可以找到循环中有哪些量是在各循环间不会变动的，提到循环外面。

强度削弱

使用执行更快的指令替代更慢的指令，例如使用 lea 和各种位移替代乘法。

删除归纳变量

归纳变量指的是循环中的一些变量，在循环的过程中每轮循环后的改变量总是固定的。如果一个循环中有多个这样的变量并且变化步调一致，则可以删除到只剩一个。

基本块的优化

为了执行局部优化常常将基本块转化为一个有向无环图。这个图中每个结点会对应一个运算并且关联一个变量的列表，结点 M 指向结点 N 表示在某条语句中，M 关联的变量的计算依赖于 N 变量的值。

这个图是随着基本块的一条条语句逐渐建立起来的。每分析到一条语句（以 x = y + z 为例），首先在图中找到关联了 x 的结点，如果找到将 x 移出这个列表；然后在图中找到关联了 y 和 z 的结点，如果没有的话就创建关联了 y_0 或 z_0 的空结点表示初始值。然后看一下有没有某个结点已经表示了 y 和 z 的相加，如果有的话直接将 x 加入到对应的列表中， 否则创建新结点。可以注意到这样的性质：每个结点关联的列表是不相交的。这个列表称为 定值变量表。

有些变量在一个基本块后面还会被用到，称为 活跃变量。对于由基本块得到的图，如果某个结点的定制变量表中所有的变量都是非活跃的，并且这个结点不被依赖，则可将其删除；重复这个过程就可以删除一部分的无用代码。

这个图中有些结点会处于特殊的 杀死 状态，这种状态的产生可能是出现了形如 x[y] = z 的数组赋值，随后所有对 x[w] 的使用，即时之前已经计算过 x[w] 的值也不能视作是公共子表达式。当处理 x[y] = z 后需要杀死所有依赖于 x 的结点，这些结点无法使用定制变量表进行优化。

图的信息和使用

从图中可以得到一些基本块的信息，例如哪些变量在赋值前被引用过；哪些计算结果可以在基本块外使用等。

使用这张图可以对基本块进行重组。由于每个结点关联了若干变量，但是它们的值是相同的，所以可以只计算出其中一个变量的值来。这个变量的选取依据是 尽量选取活跃的变量，其它的变量的值可以不求，直接使用这个变量代替；如果有多个活跃变量，应该使用赋值语句让其它变量和这个变量的值相同。
