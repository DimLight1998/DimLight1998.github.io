中间代码生成-1

中间代码生成重点在于各种语句如何进行翻译。下面两节面向的是声明语句的翻译。个人感觉这附近都是在说语法制导翻译的具体应用形式，没有多少理论上的东西了。

类型表达式

为了知道声明语句如何翻译，需要搞清楚类型和类型表达式。

类型表达式实际上表示的就是一个特定的类型。平时在高级语言里面写的 int a[50] 这种实际上是一种语法糖，a 的类型是 int[50]，类型表达式就是要让 int[50] 这种表述更加底层。

基本类型都是类型表达式，也就是说 int 和 void 这种就是一个完整的类型表达式。

复合类型可以通过更加基础的类型表达式通过 类型构造符 得到，几个常用的类型构造符如下：

  • 数组构造符：例如 int[50][20] 实际上对应着一个类型表达式 $array(50, array(50, int))$。
  • 指针构造符：例如 int** 实际上对应着一个类型表达式 $pointer(pointer(int))$。
  • 笛卡儿积构造符：我觉得这个构造符用于某些语言中的匿名元组，例如 (int, bool) 对应着 $int\times bool$，注意这个构造符在这里是中缀的。
  • 函数构造符：这个构造符接收两个类型表达式，得到一个新的类型表达式表示函数。比如 function<int(bool)> 对应着 $bool\rightarrow int$，如果有多个参数的话可以使用笛卡儿积打包。
  • 记录构造符：实际上就是结构体的构造符，使用若干笛卡儿积记录每个记录中域的名称和类型（不知道为什么要单独列出来）。

通过这些构造符和基本类型基本上可以构造出语言中出现得了的类型了。

声明语句的翻译

对于声明语句，语义分析的目的就是找到声明的变量的名字、类型、宽度，并将这些信息存放到符号表中。在语义分析过程中还能够为这些声明出的变量提供相对地址，相对地址也存放在符号表中。

每当语义分析过程中发现了一个新的变量的时候就可以在符号表中分配新的入口了，这个分配过程可以看作是某种副作用。
