数据流分析

数据流分析是一种用来获取程序执行路径上的数据流信息的方法。分析过程会将每个程序点和一个 数据流值 关联起来。这个定义过于抽象，不要深究。什么是数据流我暂且认为是 表示各个变量的值的传递过程的信息。

数据流分析有 到达定值分析、活跃变量分析 和 可用表达式分析 这几种应用。

对于语句 $s$，使用 $\mbox{In}[s]$ 表示 $s$ 之前的数据流值，$\mbox{Out}[s]$ 表示 $s$ 之后的数据流值，随着 $s$ 的不同可以得到两者之间的函数 $\mbox{Out}[s] = f_s(\mbox{In}[s])$，这个函数称作 $s$ 的传递函数，这种风格的传递函数是前向传播风格，还有一种是 $\mbox{In}[s] = g_s(\mbox{Out}[s])$ 是逆向传播风格。对于基本块 $B$ 也有 $\mbox{In}[B]$ 和 $\mbox{Out}[B]$。

对于相邻的语句有 $\mbox{In}[s_{i+1}] = \mbox{Out}[s_i]$，对于基本块 $B$，它的 $\mbox{In}[B]$ 是第一条语句 $s_1$ 的 $\mbox{In}[s_1]$，它的 $\mbox{Out}[B]$ 是最后一条语句 $s_n$ 的 $\mbox{Out}[s_n]$。对于前向风格的传递函数，有 $f_B = f_{s_n}\cdot f_{s_{n -1}}\cdots f_{s_2}\cdot f_{s_1}$；对于逆向风格的传递函数有 $g_B = g_{s_1}\cdot g_{s_{2}}\cdots g_{s_{n - 1}}\cdot g_{s_n}$。

到达定值分析

如果一条语句 $s$ 对变量 $x$ 赋值了，则称 $s$ 是 $x$ 的一个 定值。

如果存在某个路径，它以 $x$ 的某个定值 $s$ 开始，以某个程序点 $p$ 结束，且这条路径上没有对 $x$ 的其它定值，称定值 $s$ 到达 程序点 $p$。

到达定值分析的一些应用：

  • 检测循环中的不变量：一个循环中对某个变量进行了赋值，但是通过到达定值分析可能发现这个变量的值在每轮循环中都没有改变。
  • 常量合并：对某个变量 $x$ 的使用只有一个定值可以到达，并且这个定值是将某个常量赋给 $x$，则可以直接替换为这个常量。
  • 判断某个变量是否在被赋值前就被使用了。

生成与杀死定值

一个定值语句会 生成 一个定值（就是它自己），同时会 杀死 其它所有对被赋值的变量的定值。

用 $gen_d$ 表示语句 $d$ 生成的定值的集合，用 $kill_b$ 表示杀死的定值的集合。则对于某个语句 $d$ 而言，其传递函数可以写成 $f_d(x) = gen_d\cup(x - kill_d)$（自变量 $x$ 的“类型”是一个数据流值，由此看来此应用下的数据流值是一些定值的集合）。

基本块的传递函数的形式一致：$f_B(x) = gen_B\cup(x - kill_B)$，其中 $kill_B = kill_1\cup kill_2\cup \cdots\cup kill_n$，$gen_B = gen_n\cup (gen_{n - 1} - kill_n)\cup(gen_{n - 2} - kill_{n - 1} - kill_{n})\cup\cdots\cup(gen_1-kill_2-\cdots -kill_n)$。

到达定值的数据流方程

对于一个程序，它的定值被以下方程约束：

  • $\mbox{Out}[Entry] = \varnothing$，其中 $Entry$ 是在第一个基本块前的虚拟基本块。
  • $\mbox{Out}[B] = gen_B\cup(\mbox{In}[B] - kill_B)$，由传递函数的定义和计算方式得到，$gen_B$ 和 $kill_B$ 可以直接从 $B$ 中得到。
  • 对于基本块 $B$，如果它的前驱为 $A_1,A_2,\cdots,A_n$，则 $\mbox{In}[B] = \cup_i A_i$。

到达定值方程的计算

以下算法可以计算出每个基本块 $B_i$ 的 $\mbox{In}[B_i]$ 和 $\mbox{Out}[B_i]$，假定每个基本块的生成集合和杀死集合已经得到。

  1. 将包括 $Entry$ 块在内的所有基本块的 $\mbox{Out}$ 设置为空集。
  2. 对于除了 $Entry$ 的每个基本块，先使用 $\mbox{In}[B_i] = \cup_j A_{ij}$ 计算其 $\mbox{In}$（对于基本块 $B$，如果它的前驱为 $A_1,A_2,\cdots,A_n$，则 $\mbox{In}[B] = \cup_i A_i$），然后使用 $\mbox{Out}[B] = gen_B\cup(\mbox{In}[B] - kill_B)$ 计算 $\mbox{Out}$。完成后如果有任何基本块的 $\mbox{Out}$ 发生改变，则重复这一步直到没有改变。

引用-定值链

对于一个基本块中的变量的每次引用，可以为其关联一个 引用-定值链 表示所有可以到达该引用的定值：

  • 如果这次引用前，基本块中已经有了对该变量的定值，则链中只有唯一元素就是最近的定值。
  • 否则，链中元素是 $\mbox{In}[B]$ 中对该变量的定值的集合。

（考虑到到达某个基本块的路径不止一条，链中可能有多个元素。）
