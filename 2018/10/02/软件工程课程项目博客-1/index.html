软件工程课程大作业博客——后端技术的选取

一些考虑

考虑到后端的生态丰富程度和开发速度，只能从 Python 和 NodeJS 中选择。为了有更强的类型系统决定使用 TypeScript。

Web 框架打算使用 Koa，但是 Koa 本身还是比较底层，Egg.js 是对 Koa 的进一步封装，主要是引入了对 Koa 的自定义扩展的功能以及引入了一个更加强大的插件功能，暂时计划就用这个框架。

Egg.js 的一些概念

主要是对这个框架的一个 overview。

Application

Application 是一个全局的对象，整个 Web 应用中只会有一个。它可以用来发出某些事件的信号，作为开发者可以监听这些信号。

Context、Request、Response

Request 封装了一次 HTTP 请求、Response 封装了一次 HTTP 响应。Context 将一次 HTTP 请求和它对应的响应封装了起来。此外可能需要创建一个匿名的 Context，则是可以做到的。

Controller、Service

Controller 用来决定对于给定的请求（经过路由后）返回什么页面。这个过程中的逻辑可以进行抽象，抽象出来负责业务的那一部分成为 Service。

编写自己的 Controller 和 Service 时推荐的搞法是写一个类。

Cookie、Session

和 HTTP 中的一样。可以通过在 Controller 中从 Context 来获取。

Helper、Config

Helper 的实例含有一些 utility 函数，整个应用中的 utility 函数都应该被放到 Helper 中方便被复用。

Egg.js 推崇配置和代码分离，业务和框架的配置都应该被放到 Config 对象中。

Logger

Egg.js 提供了一个比较强大的日志系统，使用 Logger 类进行使用。

中间件

Egg.js 基于 Koa，它们都通过使用中间件栈来增强功能。在 Egg.js 中可以容易地编写出一个中间件，也能比较方便地在应用、框架或者路由中使用中间件。Egg.js 可以直接使用 Koa 的中间件生态。所有的中间件都具有一些通用的配置。

路由

使用路由可以将不同的请求导向不同的 Controller。路由也能完成一部份路径解析的功能，Egg 中可以使用路由解析 url 中的参数、对 url 进行分节并将传入 Controller，也能直接应用正则表达式进行解析。此外路由还能完成重定向的功能。

需要注意的是如果后面打算用 RESTful 的设计的话，Egg 在路由上提供了一些更加简单的方法。

其它

Egg 还提供了：

  • 插件：插件与中间件不同，可以把插件看作是对 Egg 本身功能的增强，例如国际化等。
  • 定时任务：框架有时需要做一些独立于任何连接的操作，这可以通过定时任务来完成。
  • 框架扩展：方便在 Application、Context、Request 和 Response 上添加方便自己的函数。
  • 启动自定义：方便在框架开启时完成自定义的一些操作。

开发中的一些概念

与 Egg 配套的 egg-bin 模块能够辅助进行开发，它含有应用启动设置、单元测试、覆盖率测试和调试相关的一些功能。
