What is Type Inference

ML 是静态类型语言，所有表达式的类型在编译时确定。类型推断是为程序中的每个表达式确定类型的过程。如果能够找到一个解使得每个表达式类型确定且相容，则推断成功，否则推断失败，可以认为程序有问题。

一般来说类型推断后再实施类型检查，但是实践中二者往往合并。

类型推断的难度取决于类型系统。但是类型系统的严格程度和类型推断的难度没有直接的关系，拒绝所有程序的类型系统和接受所有程序的类型系统实现难度是一样的。

基础类型和复合类型

基础类型是“原子”的类型，无法继续分解。

复合类型例如列表、元组和 Option。

构造复合类型的通用方法

复合类型都可以由三种方式组合地构造出：

  • Each of：包括全部的可选类型
  • One of：包含若干个可选类型之一
  • Self reference：包含自己这个类型

例如元组就是 Each of 形式的，Option 就是 One of 形式的，列表是三种形式的组合：一个 int list 可以为空（One of），也可以为一个 int 加上一个 int list（Self reference 和 One of）。

Deep Neural Networks

Deep Neural Networks

Deep L-layer neural network

将网络层数加深，具有 $L$ 层的神经网络称为 L-layer 神经网络。输入层是第 0 层，但是计算层数的时候不计入总层数。一些记号如下（和之前的大致相同）：

  • $n^{[k]}$ 是第 $k$ 层的神经元数目，故 $n^{[0]}$ 即样本的特征维数，对于二分类问题 $n^{[l]} = 1$。
  • $z^{[k]}$ 是第 $k$ 层神经元输出的未经过激活函数的值，多样本时为 $Z^{[k]}$。
  • $a^{[k]}$ 是第 $k$ 层神经元输出的经过激活函数的值，特别地 $a^{[0]}$ 是样本的特征，二分类问题中 $a^{[l]}$ 是 $\hat y$，多样本时为 $A^{[k]}$。
  • $g^{[k]}(x)$ 是第 $k$ 层的激活函数。
  • $W^{[k]}$ 是第 $k$ 层的权值，$b^{[k]}$ 是第 $k$ 层的偏移。

Neural Networks Basics

Logistic Regression as a Neural Network

Binary Classification

课程中使用的符号：

  • $m$ 表示数据集大小，$m_{train}$ 和 $m_{test}$ 分别表示训练集和测试集的大小。

  • $n$ 表示每个样本的特征向量的维数。

  • $X$ 表示整个训练集，每一列表示一个样本，相当于样本按照列进行堆叠，因此 $X$ 的规模是 $n\times m$ 的，据说这样表示更简单。

  • $Y$ 表示所有的标签，每一列表示一个样本，因此 $Y$ 的规模是 $1 \times m$ 的。

Logistic Regression

对于一个样本特征，逻辑回归用于得到这个样本对应的输出，不过这个输出被限制在了 0 到 1 的范围内，因此可以用来做一些类似二分类的事情。比如现在要对某个特征向量 $\vec x$ 进行二分类，希望得到一个输出 $\hat y$ 表示在当前特征为 $\vec x$ 的情况下真实输出为 1 的概率（二分类时这个输出是离散的，只有 0 和 1 两种情况，但是概率是连续的），可以选择 $\hat y = \vec w^T\vec x + b$ 这个模型（这时是线性回归），但是这样得到的值不一定在 0 到 1 之间，因此可以套上一个 sigmoid 函数 $\displaystyle{\sigma(x) = \frac{1}{1 + e^{-x}}}$，选择模型 $\hat y = \sigma(\vec w^T\vec x + b)$，此时为逻辑回归。

代码生成

代码生成器的主要任务

代码生成器将中间表示转换到汇编语句。

三个主要任务：

  • 选择适当的目标机指令表示中间表示
  • 对寄存器的分配和指派
  • 确定指令的执行顺序

一个简单的目标机模型

为了后续方便引入一个简单的汇编语言，它的指令有：

  • LD x, y：读入 y 至 x
  • ST x, y：将 x 写入 y
  • OP x, y, z：计算 y OP z 后存入 x
  • BR l：无条件跳转到 l
  • Bcond c, l：在满足条件 c 时跳转到 l

以及各种寻址方式。

指令选择

这一节主要是几个例子，表示对于给定的中间表示如何生成出对应的指令。显然没有什么好记录的。

寄存器的选择

寄存器分配

当要为 x = y op z 分配寄存器的时候，调用 getReg 函数选择寄存器。

假设为 y 选出的寄存器为 $R_y$，如果其中存放的不是 y，就生成一条 LD R_y, y。对 z 也是同理。

现在 $R_y$ 和 $R_z$ 中存放的是正确的值，生成对应的汇编语句即可。

使用到的数据结构

为了实现 getReg 需要维护两张表。一张是记录每个寄存器中存放了哪些变量（一个寄存器中可能不止存放了一个变量，也可能存放了多个相同的变量），称为 寄存器描述符，另一张记录每个变量被存放在哪些位置（是寄存器、内存、栈等空间的组合），称为 地址描述符，第二张表可以在符号表中实现。

在基本块结束时，如果某个变量不在自己的内存区域中，需要生成一条保存语句。这个类似于缓存 dirty 的策略，可以加速程序运行。

管理寄存器描述符和地址描述符

LD R, x 指令

  1. 修改 R 的寄存器描述符为只包含 x；
  2. 将 R 加入到 x 的地址描述符中；
  3. 将 R 从其它变量地址描述符中移除。

OP R_x, R_y, R_z 指令

  1. 修改 R_x 的寄存器描述符为只包含 x，从其它寄存器描述符中删除 x。
  2. 修改 x 的地址描述符为只包含 R_x，从其它变量地址描述符中移除 R_x。

其它语句按照类似方式得到。

特别地，对于复制语句 x = y 只用分配一个寄存器 R，R 对应的寄存器描述符中可以既存放 x 也存放 y。

寄存器选择函数 getReg 的设计

要对一个变量选择寄存器时，按照如下流程：

  1. 检查变量是否已经在某个寄存器中了，如果是，返回这个寄存器。（不管是读取还是写入都是对的！）
  2. 检查是否有空的寄存器，有则返回。
  3. 否则，计算出每个寄存器被使用后的“代价”，这个代价和每个寄存器中存放的变量数量以及后续的使用情况有关。选出代价最小的寄存器返回。

getReg 的输入应该是指令而不是变量，这样可以对形如复制语句的语句作特殊处理，为两个变量选择相同的寄存器。

有一些特殊的情况，例如 x = y + z 中如果确认了 y 在以后不会被使用，那么为 x 选择寄存器时那个 只存放了 y 的寄存器对其而言是空寄存器。
