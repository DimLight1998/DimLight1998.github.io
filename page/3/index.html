SQL 语言之复杂查询与视图

In 子查询

子查询

子查询存在是为了应对查询中和集合有关的查询，例如元素是否属于某集合、集合是否为空等。

子查询返回的是一个集合。

In 子查询

判断一个集合是否在给定子查询中：$$Expression [not] in $$SubQuery。一般在 where 子句中出现。

在有子查询的情况下，查询可以分为外层查询和内层查询。如果内层查询不依赖外层查询（不使用外层查询的信息），称为 非相关子查询，否则是相关的。相关子查询中，信息只能从外层向内层传递，不能反过来。

Theta Some/All 子查询

这个查询主要判断集合之间的包含关系。假设 θ 表示一个二元比较关系，例如等于、大于、小于等于等。那么有两种 theta 查询：$$Expression θ some $$SubQuery 和 $$Expression θ all $$SubQuery。前者在子查询中某个值满足 θ 关系时就能成立，后者需要在子查询的所有值都满足关系的时候才成立。一般在 where 子句中出现。

  • $$Expression in $$SubQuery 可以看做是 $$Expression = some $$SubQuery。
  • $$Expression not in $$SubQuery 可以看作是 $$Expression <> all $$SubQuery。

SQL 语句主要由以下引导词引导：

关系模型之关系代数

关系代数都是对集合进行操作的：输入和输出都是集合，从已有集合得到想要的集合。

下面的操作按照难度递增，分别是基本操作、扩展操作和复杂扩展操作。

什么是关系代数

关系代数通常以一个或多个关系作为输入，结果是新的关系。关系代数是一种抽象的语言。

关系代数可以分成 集合操作（并交差积） 和 纯关系操作（投影、选择、连接、除）。

DBMS 可以先解析 SQL 语言，将其翻译为多个关系代数操作的序列。这样，只要实现了基本的一些关系代数操作就可以实现一个 DBMS 了。

并相容性的概念

两个关系是并相容的，如果它们的属性数目相同，相同位置上的属性的类型也相同。

简单来说，两个关系并相容，表示可以把一个关系纵向拼接到另一个关系下面形成更大的关系。

并相容性是其它很多操作的基础。

什么是关系元组演算

关系运算分为关系代数（见上次的笔记）与关系演算。关系演算是以谓词演算为基础的，按照谓词变量不同分为 元组演算 和 域演算。从名字可以看出一个是针对行，一个是针对列。

后面可以看到在一定的条件下，关系代数和关系演算是可以相互转化的。
