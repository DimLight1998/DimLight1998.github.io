什么是数据库

数据库：有相联关系的数据的集合。

表：按行按列组织和展示的数据。

表的一些术语

下面定义中的 表 指的是包含每列标题的表。

  • 表名：一张表的名称。
  • 表标题（格式）：每列的标题。
  • 表内容：表中除去表标题剩下的数据。
  • 记录（元组）：一行数据。
  • 字段（属性、数据项）：一列数据。一列数据又分 列名 和 列值。
  • 关系模式：表名和表标题。
  • 关系：不严格地讲，关系就是整张表。

由此可以重新定义数据库：相互之间有关联关系的表的集合。

关系数据库是结构化的数据库。存在非结构化的数据库。

什么是数据库系统

数据库系统是以下软硬件的合称：

  • 数据库（DB）：存储的实际数据。
  • 数据库管理系统（DBMS）：管理数据库的系统软件，例如 MySQL。
  • 数据库应用（DBAP）：可以想成是访问、修改数据库的客户端，可以完成特定的业务功能。
  • 数据库管理员（DBA）：负责管理数据库和使用 DBMS 的人员。
  • 计算机基本系统：基本硬件设施。

从用户角度看数据库管理系统

在 DBMS 的使用者看来，DBMS 应当提供以下功能：

  • 允许用户创建表：DBMS 提供了 数据定义语言（DDL），可以用来创建表。
  • 允许用户操纵数据：DBMS 提供了 数据操纵语言（DML），可以用来进行增删改查。
  • 允许用户控制客户的权限：DBMS 提供了 数据控制语言（DCL），可以用来控制 DBA 的访问权限。
  • 提供一些实用程序：例如转储、性能监控等。

DDL、DML、DCL 合起来就构成了 SQL。

SQL 可以被嵌入到其它高级语言中，被嵌入的语言称为 宿主语言。

从系统角度看数据库管理系统

在 DBMS 的实现者来看，DBMS 完成了以下事情：对于各种语言（例如 DDL），DBMS 会先使用一个编译器得知该语言的语义，然后使用对应的执行引擎操作文件完成任务。不同语言的编译器和执行引擎都不同。

DBMS 可以经由操作系统工作，也有越过操作系统进行底层控制的 DBMS。

DBMS 由很多更小的程序构成以完成工作：

  • 语言编译器
  • 查询优化与查询实现：主要用于提高查询的性能
  • 数据存储与索引：运用各种缓冲和索引
  • 通信控制：提供网络环境下的操作与传输功能
  • 事务管理：提供可靠性与一致性
  • 故障恢复
  • 安全性控制：防止出现授权问题
  • 完整性控制：保证数据完整、正确
  • 数据字典管理
  • 应用程序接口
  • 其它
