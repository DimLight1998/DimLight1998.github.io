嵌入式 SQL 语言之基本技巧

本讲内容已经过时，因此笔记粗略。

什么是嵌入式 SQL 语言

普通的 SQL 称为交互式 SQL，存在学习成本，并且过程性差。可以将交互式 SQL 嵌入到一些高级语言（称为宿主语言）中完成更加复杂的功能，这就是嵌入式 SQL。

这样做会引入很多问题，例如高级语言如何和数据库进行交互等。

程序与数据库连接

要解决的问题：

  • 宿主语言如何和数据库连接和断开。
  • 宿主语言如何提交和撤销操作。
  • 宿主语言如何将变量传递给 SQL（可以想成 SQL 现在是一个需要被插值的模板字符串，插值来源是数组语言的变量）。

为什么需要提交和撤销

事务 在 DBMS 使用者视角上指的是多条 SQL 语句的一次执行；在 DBMS 实现者视角上指的是一组需要要么全部执行、要么全部不执行的 SQL 语句。

事务的特性：

  • 原子性：要么全部执行，要么全部不执行。
  • 一致性：事务的操作状态是正确的，一致的。
  • 隔离性：多个事务的执行结果和其执行顺序无关，即使它们是并发执行的。
  • 持久性：已提交事务的影响是持久的，被撤销事务的影响是可恢复的。

事务处理是 DBMS 的核心技术。

为什么需要游标

要解决的问题：

  • SQL 语句如何执行。
  • 如何将 SQL 检索到的结果返回给宿主语言。
  • 如何将 SQL 语句中的常量改为变量。

如果检索结果只有一行，那么可以直接送到宿主语言的变量中。如果检索结果有多行则需要使用 游标 来对检索结果进行逐行的访问。游标实际上就是检索结果的迭代器，每次读取一行。

游标应用示例

本节无笔记。

可滚动游标

标准的游标是单向的，只能从检索结果开始读取到结束。一条已经遍历过的数据如果希望再次访问只能重新打开一个游标。

通过引入可滚动游标可以让游标双向移动，并可以跳转到第一条记录或最后一条记录，但是很多 DBMS 没有支持这一特性。

利用游标进行数据库增删改

对于记录的删除和更新，有两种方式。一种是和交互式 SQL 类似的 查找删除/更新，此时嵌入式 SQL 的写法和交互式 SQL 写法很类似；另一种是 定位删除/更新，此时删除或者更新的是当前游标对应的那条记录。

对于记录的增加则只能像交互式 SQL 那样做。
