SQL 语言之复杂查询与视图

In 子查询

子查询

子查询存在是为了应对查询中和集合有关的查询，例如元素是否属于某集合、集合是否为空等。

子查询返回的是一个集合。

In 子查询

判断一个集合是否在给定子查询中：$$Expression [not] in $$SubQuery。一般在 where 子句中出现。

在有子查询的情况下，查询可以分为外层查询和内层查询。如果内层查询不依赖外层查询（不使用外层查询的信息），称为 非相关子查询，否则是相关的。相关子查询中，信息只能从外层向内层传递，不能反过来。

Theta Some/All 子查询

这个查询主要判断集合之间的包含关系。假设 θ 表示一个二元比较关系，例如等于、大于、小于等于等。那么有两种 theta 查询：$$Expression θ some $$SubQuery 和 $$Expression θ all $$SubQuery。前者在子查询中某个值满足 θ 关系时就能成立，后者需要在子查询的所有值都满足关系的时候才成立。一般在 where 子句中出现。

  • $$Expression in $$SubQuery 可以看做是 $$Expression = some $$SubQuery。
  • $$Expression not in $$SubQuery 可以看作是 $$Expression <> all $$SubQuery。

Exists 子查询

这个查询主要判断集合是否为空。语法是 [not] exists $$SubQuery。它一般出现在 where 中。一般的使用方式是使用相关子查询，这样子查询就会包括外层的信息，于是 exists 查询就有用了。例如下面例子中的 S# 来自外层查询：

用 SQL 处理空值

在有空值的情况下会影响到聚集函数的语义。

SQL 中可以使用 is [not] null 来判定一个值是否为空。

空值的处理：

  • 空值不满足任何查找条件，除了 is null。
  • 参与算术表达式时表达式结果为空值。
  • 参与比较运算时总是返回 false。
  • 除了 count(*) 之外的所有聚集函数都忽略空值所在记录。

用 SQL 表达连接与外连接操作

连接类型有四种：内连接、左外连接、右外连接和全外连接，分别是 inner join、left outer join、right outer join、full outer join。

连接方式有三种：自动的自然连接、带条件的连接和指定属性的自然连接。自动自然连接就是数学含义上的自然连接，所有属性名相同并且值相同就会发生连接；待条件的连接需要给出两个关系中不同记录需要满足什么条件才会连接；指定属性的自然连接也是自然连接，但是属性不是所有属性，而是用户指定的。
