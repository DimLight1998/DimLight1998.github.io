TaPL 读书笔记 5 Recursive Types

发表于 2019-05-08 | 更新于 2019-05-31

笔记对应原书 20-21 章。

上一部分（Subtyping）的重点回顾：

  • 简单的类型系统表达能力不够，为了让类型之间能够“兼容”，需要引入类型之间的 subtyping 关系，让一种类型能够“替代”另一种类型出现。Subtyping 关系构成类型上的偏序关系。
  • 引入 subtyping 后类型系统依然安全，表达能力没有增强，可以通过 coercion semantics 将带有 subtyping 关系的语言翻译到简单类型语言。
  • 可以引入 所有类型的子类型 Bot 和 所有类型的超类型 Top，增强表达能力。Top 包括所有的值，Bot 不包括任何值，用来表示不可返回的计算。
  • 两个类型的 meet 是它们的 intersection type，两个类型的 join 是它们的 union type，这些类型的取值对应的是原来两个类型取值集合的交和并。
  • 引入了 subtyping 的类型系统会在语言中定义 subtyping rules，其中会包括 subtyping 关系的自反性和传递性，以及语言内建的一些 subtyping 关系（例如函数和记录）。在有自反性和传递性的情况下，两个类型的 subtyping 关系的判定不容易实现，因此引入了 algorithmic subtyping 讲自反性和传递性直接“集成”到语言内建 subtyping 关系中，这样 type checker 就容易实现了。
  • 使用 lambda 演算可以对 OOP 进行编码来证明其性质，但是研究 OOP 的类型系统时通常会搞个新的演算系统。

在引入了 subtyping 后类型系统的表达能力得到了增强，但是还是有些事情搞不定：

  • 没法表达“列表”这种结构
  • 没法表达 fix 不动点组合子的类型

这之前，我们通过扩展语言的核心语法完成了上面的事情。在不修改核心语法的情况下，需要引入 recursive types。比如自然数列表的类型实际上是 NatList = <Nil: Unit, Cons: {Nat, NatList}>，为了不用等式描述这个类型，它的另外一种写法和 lambda 演算中的函数抽象类似：μX.<Nil: Unit, Cons: {Nat, X}>。你可以将这个类型展开成一棵树，叶子结点都是 base type。这棵树的高度是无限的。

引入了 recursive type 之后，一个说明 recursive type 有多强的例子是 所有的无类型 lambda 演算的 term，都可以编码为含有 recursive type 的 lambda 演算中的 μX.X -> X 类型下的一个 term。

对于两个 recursive type 是否相等，有两种定义：equi-recursive 和 iso-recursive。区别在于前者在比较时会自动“展开”一个 recursive type 进行比较，比如对于 NatList 的 term t 以及 <Nil: Unit, Cons: {Nat, NatList}> 的 term s，它认为 t 和 s 来自同一种类型，不需要其他条件；而对于 iso-recursive 就需要显式地用 fold 和 unfold 两个过程来折叠或者展开定义，它认为：

  • t 和 s 不是来自同一种类型
  • unfold[NatList] t 和 s 来自同一种类型
  • t 和 fold[NatList] s 来自同一种类型

后面一种的 type checker 要容易写一些。

最后一个问题就是含有 recursive type 的情况下 subtyping 如何进行判定？比如有两个 process，类型分别为 μX.Nat -> {Even, X} 和 μX.Even -> {Nat, X}，已知 Even 是 Nat 的子类型，应用一下元组和函数的子类型规则就知道前面的 process 的类型是后面的 process 的类型的子类型，但是要搞出一个通用的算法还是比较复杂（书中后面讲 recursive type 都是在证明这个算法）。

以下是 recursive type（equi-recursive） 中 subtyping 的判定的一些理论。

对于一个全集 $\mathcal U$，定义一个 generating function $F:P(\mathcal U)\mapsto P(\mathcal U)$。如果 $A\subseteq B\subseteq \mathcal U \Rightarrow F(A)\subseteq F(B)$，则称 $F$ 是 单调 的。下面所有的函数全部默认是单调的。

对于集合 $X\subseteq \mathcal U$，如果：

  • $F(X)\subseteq X$，则 $X$ 是 $F$-closed 的
  • $X\subseteq F(X)$，则 $X$ 是 $F$-consistent 的
  • $F(X) = X$，则 $X$ 是 $F$ 的不动点

Knaster–Tarski 定理 指出，$\mathcal U$ 的所有 $F$-closed 子集的交是 $F$ 的最小不动点，记作 $\mu F$；所有 $F$-consistent 子集的并是 $F$ 的最大不动点，记作 $\nu F$。

由此得到两个显然的推论，分别是 induction 和 coinduction 背后的原理：

  • （induction）如果 $X\subseteq\mathcal U$ 是 $F$-closed 的，则 $\mu F\subseteq X$
  • （coinduction）如果 $X\subseteq\mathcal U$ 是 $F$-consistent 的，则 $X\subseteq\nu F$

上面这些都是理论基础，和 recursive type 还没有关系。为了表示 recursive type 我们可以使用前面的 $\mu X.T$ 的表达方式，但是这里先用树状结构进行表达（即将类型展开为树，叶子为 base type，结点为各种类型的构造子，儿子之间有序），这样的类型叫做 tree types。

Generating function 会将 $\mathcal U$ 中的子集映射到另一个子集。对于任意的 $x\in\mathcal U$，可以找到 $P(\mathcal U)$ 的子集 $G_x$，使得 $\forall s\in G_x, F(s)\ni x$。对任意 $x$，如果以下条件之一总是满足：

  • $G_x$ 是空的（说明不存在任何 $\mathcal U$ 的子集 $X$ 使得 $x\in F(X)$）
  • $G_x$ 中有个最小元素（这个元素可能是 $\varnothing$，表示 $x\in F(\varnothing)$）
