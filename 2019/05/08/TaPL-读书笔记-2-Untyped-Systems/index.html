语言的语义

程序的语义指的是 term 如何被求值。程序的运行过程实际上是对 term 的变换过程，最终的目的是将 term 变换到一个 value。所有 value 是所有 term 构成集合的一个子集，该子集是人工定义出的。程序的语义提供了 term 转换为 value 的过程中每一步的依据。

语义的描述方式有以下几种（我只看懂了操作语义）：

  • 操作语义：操作语义给出了一个状态机以及状态变迁的规则，程序的执行即为状态的变迁。对于简单的语言来说，term 就可以作为状态。
  • 指称语义：指称语义对 term 的解释是将其映射到一个 semantic domain 上，这个 domain 中的东西都是一些数学对象。
  • 公理语义：这个就更加抽象了，将规则本身作为语言的定义，将 term 的性质作为 term 的定义。

操作语义又分为 小步语义 和 大步语义：小步语义给出了 term 计算到 value 过程中每一步的规则，而大步语义直接给出了怎么到最后的结果。

书中用到的所有语义都是操作语义。

求值过程中的一些性质

确定性

在上述玩具语言中，语义定义有个性质是：$t\rightarrow t_1\land t\rightarrow t_2\Rightarrow t_1 = t_2$（可以证明）。它的意思是 你只能按照一种顺序对 term 应用规则，就是说 term 的求值过程总是固定的。这个性质叫做 determinacy，并非所有语言都满足。

Normal Form

如果对于某个 term $t$，任何规则都不能找到对应的 $t'$ 使得 $t\rightarrow t'$，则称 $t$ 是一个 normal form。

在一般的语言中，所有的 value 都是 normal form，但不是所有的 normal form 都是 value。不是 value 的那些 normal form 无法继续进行运算，这样的状态称为 stuck。对于一般的语言而言，并非所有的 term 都可以被求到 value，有些会在求值过程中 stuck。

无类型 lambda 演算

复杂的语言都可以看作是一个简单的核心和对核心的扩展。这些扩展可以用核心来表示。Lambda 演算就是这样的一个核心。（其它核心包括对并发建模的 pi-calculus 和对对象建模 object calculus 等。）

Lambda 演算的语法与语义

从上面的笔记中可以知道，语言应当定义 term、value 以及 term 的变换规则。

Lambda 演算的 Term

Lambda 演算的 term 只有三种：变量、函数定义和函数应用。其中，变量指的是函数定义时用到的参数。

Lambda 演算的 Value

Lambda 演算中所有的东西都是函数，函数的参数和返回值也都是函数。所以 Lambda 演算中的 value 就是函数。

关于语法的其它东西

对 term 的定义实际上定义的是 抽象语法，它与语言的表现形式无关，可能是以 AST 的形式存在的。但是更实用的是 具体语法，例如如何使用文本将语言表示出来。

Lambda 演算的具体语法中的一些规则：

  • 函数应用是左结合的：$a\ b\ c\ d = (((a\ b)\ c)\ d)$
  • 函数抽象中函数体的范围尽可能大：$\lambda x.\lambda y.xyx = \lambda x.(\lambda y.(xyx))$

柯里化

Lambda 演算中一个函数只接受一个参数，但是可以通过柯里化来表示接受多个参数的函数。

Free Variable 与 Bounded Variable

Term 中的变量，如果是项中某个函数抽象的参数，就称为是 bounded variable，否则是 free 的。$\lambda x.t$ 这个 term 中的所有 $x$ 都是 bounded 的。

例如 $\lambda x. x y$ 中，$x$ 是 bounded 的，$y$ 是 free 的。

没有 free variable 的 term 称为 组合子。

求值顺序

由于求值规则会决定求值顺序，因此先看下 lambda 演算中有哪些常用的求值顺序。

  • Full beta-reduction：任意选择 redex 求值。这种求值策略顺序不是唯一的。
  • Normal order：每次选择最左边，最外层的 redex 进行求值。

上面两个策略不是很常用，更加直观、常用的策略是：

  • Call by name：在函数应用时，将函数体中的形参替换为实参。先进行替换，再对参数求值。这种策略是惰性的。
  • Call by value：在函数应用时，先对实数求值，等求完后再替换掉函数体中的形参。这种策略是严格的。

因为使用 call by name 时可能导致实参被多次求值，像 Haskell 这样的语言用的策略是优化版本的 call by name，叫做 call by need——如果参数没有被用到就不会被求值，如果被用到了，就求值并将结果保存起来，再使用时直接用这个保存的结果。

无类型 Lambda 演算扩展

可以看到，原生 lambda 演算可以表示布尔代数和自然数，因此可以 扩展 lambda 演算系统，让它原生支持这些东西。

例如，可以引入 if 表达式，就能写出 $\lambda x.\text{if}\ x\ \text{then}\ y\ \text{else}\ z$ 这样的函数了。

当对 lambda 演算进行扩展之后，这些添加的内容就变成了 语言的一部分。此时要添加新的语法和语义。添加这些东西没有让 lambda 演算能力增强，但是让它表达能力更强了（新系统能做的，原来的系统都能实现，只是比较麻烦）。
