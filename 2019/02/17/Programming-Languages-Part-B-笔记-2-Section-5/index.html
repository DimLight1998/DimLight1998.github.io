Syntax and Parentheses

Racket 的语法很简单，只要几句话。Racket 中的一切都是以下几种之一：

  • Atom：例如 #t 和 #f（其他语言的 true 和 false），34，"str"，null，4.0 和变量 x。
  • Special form：例如 define、lambda 和 if，可以使用宏来扩展。
  • Sequence：(t0 t1 t2 ... tn)，如果 t0 是 special form 那么整个序列就有特殊含义，否则就是函数调用。

Racket 中的小括号可以和中括号混用——所有可以使用小括号的地方都可以使用中括号，但是有特定的习惯。

这种语法的一个好处是将语言转换为抽象语法树是很容易的，基本上每个括号代表一棵子树。

Parentheses Matter! (Debugging Practice)

Racket 中额外的括号可能是错误的。(e) 表示以 0 个参数调用 e 的结果，((e)) 表示以 0 个参数调用 e 的结果，对调用的结果，再次以 0 个参数调用之。每个括号如果不是上节提到的 special form 的话，都表示一次函数调用。

如果写出了这种错误的代码，或者对于函数调用了错误数量的参数，这种错误只有在运行到对应的语句时才会表现出来（例如写出了 (1 + 2)）。不过对于一些 special form 来说在编译阶段也能确定错误，例如写出了 (if #t + 1 2 3) 就是错的，因为 if 开头的 sequence 必须有 4 个元素，正确写法应该是 (if #t (+ 1 2) 3)。

Cond

cond 语句相当于多个 if-else 语句。它需要多个条件顺次进行判断，一旦其中一个满足，就会对相应表达式求值。语法为 (cond [cond_1 expr_1] [cond_2 expr_2] ... [cond_n expr_n])。一般建议将 cond_n 设定为 #t，否则 Racket 会返回一个空类型的数据。语句中的中括号为习惯。

Toplevel Bindings

在文件中顶层建立的绑定在语义上类似于 letrec，可以对使用后面绑定的内容，但是求值是按照顺序进行的。

Racket 中对于 set! 的限制

每个文件都是一个模块。模块中的内容如果在模块内部没有被 set! 过，那么无法在模块外部 set! 它们。语言自带的例如 +，-，cons 等都位于一个没有 set! 它们的模块中。

编程中强烈不推荐使用 set!。

mcons For Mutable Pairs

Racket 中的列表中的元素默认是不可改变的，这样设计的好处有：

  • 防止某个变量绑定为列表的子列表时列表发生变动导致该变量也变动
  • 可以以常数时间计算出列表长度，只有 proper list 能这么做，如果列表可变的话可能会把 proper list 转换成 improper list

Racket 中存在可变元素的列表，用以下函数进行构造和修改：

  • mcons 建立可变列表。
  • mcar 访问可变列表的头。
  • mcdr 访问可变列表的尾。
  • mpair? 询问是否是一个可变列表。
  • set-mcar! 将列表的头改变为指定元素。
  • set-mcdr! 将列表的尾改变为制定元素。

其中 set-mcdr! 可对一个子列表进行操作，所以如果需要改变 a 的第二个元素的话可以用 (set-mcar! (mcdr a) x)。

Using Streams

Stream 是一种 idiom，表示无穷长的序列。这种序列是没法显式地写出来的。

用一个 thunk 表示一个 stream。对这个 thunk 求值后会得到一个二元组，第一个元素是该 stream 的第一个值，第二个元素又是一个 thunk。如果调用该 thunk 会得到一个二元组，第一个元素是 stream 的第二个值，第二个元素是可以从中求出第三个值的 thunk。依此类推。第 k 个值会从某个 thunk 中求出，这个 thunk 求值后得到的是一个二元组，第一个元素是第 k 个值，第二个元素是可以求出下一个值和下一个 thunk 所组成二元组的 thunk。

Macros: The Key Points

使用宏可以为语言扩展新语法，添加语法糖。宏实际上是指示如何将新的语法翻译到旧的语法上。宏的展开一般在编译之前进行。

Racket 的宏系统设计较好，比 C、C++ 的宏系统靠谱。

Tokenization, Parenthesization, and Scope

这节是一个评价宏系统的三个方面。如果宏系统在这些方便处理得较好，用起来就比较爽。

Tokenization

宏展开实际上是查找替换的过程，但是这种查找是以 token 为单位的，不是以字符串为单位的。也就是说，如果将 head 定义为 car 的宏，那么 cart 不应该被替换为 headt。

Parenthesization

C、C++ 中的宏需要考虑到括号的问题，因此容易写出 #define ADD(x, y) ((x) + (y)) 这样的代码以防止运算优先级出现问题。但是 Racket 的宏系统已经考虑到了这一点。实际上因为 Racket 的语法简单而统一，宏系统的括号问题很容易得以优雅地解决。

Scope

Racket 中的宏定义拥有自己独立的 scope。
