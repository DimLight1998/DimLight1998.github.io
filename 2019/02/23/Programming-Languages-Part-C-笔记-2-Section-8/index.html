Introduction to Ruby

Ruby 的一些特性：

  • 所有的值都是对象，是纯 OOP 的。
  • 是基于类的（存在不基于类的语言，比如 JavaScript）。
  • 是动态类型语言。
  • 方便进行反射：能够在运行时查看对象属性。
  • 能在运行时改变类。
  • 闭包是一种常用的 idiom。
  • 是一门脚本语言。

Classes and Objects

Ruby 中的所有值都是某个对象的引用，包括其它语言中的原生类型，所以可以执行像 5.abs 这样的操作来求 5 的绝对值。

对象拥有自己的状态（Ruby 中所有的状态都是私有的）和自己属于的类，这个类决定了自己的行为，彼此之间通过类的方法进行交流（调用某个对象的方法可以看做是 向对象传递消息）。

定义类

定义或调用类的方法时参数用括号括起，逗号分割。如果只有一个或零个参数可以省略括号。

使用 ClassName.new 构造新对象。

self 关键字

使用 self 关键字访问当前实例。如果在类中调用当前实例的方法，也可以省略 self，作为语法糖。

Object State

Instance variables

Ruby 中每个对象有自己的变量（称作状态），这些变量都是私有的，只能通过函数来访问。这些变量叫 instance variables。

Instance variables 通过 @ 前缀定义和访问，例如 @foo。访问未定义的私有变量会返回 nil 但是不报错。

Aliasing

x = y 会导致 x 和 y 指向同一个对象。对 x 状态的改变也会影响 y 的状态，因为实际上是同一个对象。

Initialize

可以为类定义 initialize 方法，该方法可以接受参数。initialize 方法会在构造对象时被调用。常见的用法是在 initialize 方法中初始化各个 instance variable，防止后续访问时返回 nil。由于这种用法不是语言定义的一部分，因此和其它 OOP 语言不同，Ruby 中同一个类的不同对象可能有不同的 instance variables。

Ruby 中的方法可以有默认参数。

Class variables

类可以有自己的变量，这种变量通过 @@ 前缀进行定义和访问，例如 @@bar。

Class constants and methods

类可以有常量，常量通过大写首字母进行定义，例如 Bazz。常量不能被更改，在类外通过 ClassName::Bazz 进行访问。

类方法在定义时的语法为 def self.method_name(args) ... end，调用语法为 ClassName.method_name(args)。类方法中只能访问类变量、类常量和其它类方法。

Method visibility

方法默认访问性是 public 的。

可以使用 public、protected、private 关键字指明方法的可见性。一旦使用，在该关键字到下一个关键字之间的所有方法都会拥有这种可见性（与 C++ 类似，与 Java 不同）。

A Longer Example

  • 使用 raise "something as string" 抛出异常；
  • 使用 if ... elsif ... else 进行多分支判断；
  • 大部分类型可以使用 to_s 方法将自己转为字符串，这是 convention；
  • 使用 e1 if e2 语句在仅当 e2 为真的情况下执行 e1；
  • "2 is #{if 2 > 1 then "greater" else "less" end} than 1"，使用 #{...} 进行字符串插值。
  • 如果某个方法会改变对象状态，这个方法命名习惯以 ! 结尾。
  • 如果一个方法名称为 +，例如 def + r，那么调用的时候 a.+ b 可以写为 a + b，这是语法糖。
  • Ruby 中可以在顶层定义函数。

Everything is an Object

Ruby 中大部分的“东西”都是对象。几乎所有的操作都是在调用某个对象的方法。例如 3 + 4 实际上调用的是 3 这个对象的 + 方法。

nil 也是一个对象，它有 nil? 等方法。除了 nil 之外的所有对象在调用 nil? 时都返回 false。Ruby 中进行判断时，只有 nil 和 false 两个值被认为是假的。

Ruby 中所有的类都继承自 Object 类，顶层定义的函数被看做是向 Object 类中添加了新的方法。所以顶层函数实际上还是对象的方法。

每个对象都有一些共有的方法（来自 Object 类），例如 methods 方法会列出这个对象的所有方法，class 对象会给出这个对象所属的类。这些函数可以用来进行反射。Ruby 中的“类”也是一个对象，这个对象所属的类是 Class 类，Class 类也继承自 Object 类。（Object 的父类是 BasicObject，BasicObject 的父类是 nil。）

总结：Ruby 中只有对象和对方法的调用。

Class Definitions are Dynamic

Ruby 中类的定义是可以动态改变的，可以在运行时向其中添加新的方法或修改已有的方法。一旦修改完成，该类的所有对象都会使用新的定义，不管它们是否在修改前就被构造出来了。这么做可行的理由是 Ruby 中的类也是一个对象，对象可以被修改，而类则定义了示例如何工作。

这么做实际上破坏了程序的抽象性，但是有时候很方便，例如想要在一个类中扩展出一个方法。

Arrays

Ruby 中的数组（Array 类）是最常用的线性结构。数组中元素类型不要求一致。

它的使用方式比较灵活，下面是一些使用方式：

  • 使用 a[i] 进行读取，a[i] = b 进行修改，越界时会得到 nil 但是不报错（和访问未定义变量一样）；

  • 使用负的下标从末尾开始索引（与 Python 类似）；

  • 当为越界的元素定了值时，数组会自行扩展（与 JavaScript 类似）：

  • 使用 + 连接两个数组；

  • 使用 | 可以得到两个数组中不重复的元素：[3, 2, 3] | [1, 2, 3] 的结果是 [3, 2, 1]；

  • 使用 Array.new(size) 可以得到大小为 size 的空数组，元素全部是 nil。

  • 使用 push 和 pop 可以把数组当栈用，栈底是数组头；

  • 使用 push 和 shift 可以把数组当队列用，队尾是数组尾，使用 unshift 可以在头部插入元素；

  • 数组可以使用 [index, offset] 方式的索引来进行切片，并且可以对切片赋值，结果会影响到原数组；

  • 调用用数组的 each 方法相当于做 map 操作，语法上需要用到下一节的内容。

注意数组的 alias 问题。如果 a 是数组，b = a 会得到相同指向的数组，b = a + [] 会创建新数组。

Blocks

Ruby 中的 Block 是很有趣的一个特性。对于所有的方法，在调用时都可以传入 0 个或者 1 个 block，但是具体怎么使用取决于要调用的函数。

Block 和闭包很像，可以用来传递匿名函数，并且使用词法作用域。

语法：使用 { e1 } 构建不带参数的 block，{ |x, y| x + y } 构建带参数的 block。如果有多行，推荐将 { 和 } 替换为 do 和 end。

示例：

  • 3.times { puts "hi" }
  • [4, 5, 6].each { |x| puts x }

  • map/collect 会执行 map 操作（会得到新的数组）
  • inject 会执行 foldl 操作
  • select 会执行 filter 操作
  • any? 和 all? 询问数组中是否存在某性质的元素或者全为某性质的元素
  • count 统计符合性质元素个数

通过使用 block、这些函数以及 range 特性（后面会写）可以避免编程时使用循环。

Procs

First class expression

Ruby 中，block 不是对象。所谓的 first class expression 指的是满足可以作为表达式的结果、可以被传递给函数、可以放在数组中等性质的那些表达式，但是 Ruby 中 block 不能这么用。不是 first class expression 的表达式是 second class expression。

Ruby 中的 block 的使用方式受限于上一节的那种情况，虽然大部分情况下使用高阶函数都是这么用的，但是 block 比 closure 要弱一些，比如它没法存到一个数组中。

为何这样设计

因为大部分情况下，使用高阶函数的模式都是只传入一个闭包，并且要将闭包存起来的需求较少，这就有两种不同的设计：

  • 将函数作为 first class 的，但是使用的时候可能会带来不便
  • 将函数作为 second class 的，但是通过设计语法让使用高阶函数更加简单，代价是当有复杂的需求时要对 second class 的东西进行封装

Ruby 选择第二种方式，将使用高阶函数的常用模式作为语言的一部分。

Subclassing

Ruby 中继承的语法：

如果不指明，则继承自 Object 类。继承后可以得到父类中的所有 public 和 protected 方法。Ruby 中的继承不会得到父类的变量，因为它们都是私有的，也因为它们都是动态创建出来的。子类中是否会拥有这些变量取决于父类的相应函数是否会执行。

在子类的 initialize 函数中可以通过调用 super 函数访问父类的构造函数。super 也可以用来调用父类同名函数（如果在子类中被覆盖的话）。

super 这里会将函数的参数传给父类同名函数的参数，也可以手动指定。

在继承自 Class 对象的函数上调用 superclass 方法可以获取到父类类型。

  • 使用 is_a? 方法可以判定某个对象是否是某个类的实例，或者它的类型是否继承自某个类。

  • 使用 instance_of? 方法可以判定某个对象是否是某个类的实例。

使用这些判断会失去鸭子类型的优势。

Why Use Subclassing?

使用继承可以复用代码；将子类对象看作是特化的父类对象也是有好处的。但是滥用继承有时候也不好。

在不使用继承的情况下要实现类似的效果，可以：

  • 直接修改原始类的代码。这在 Ruby 中是可行的（见前面的笔记），然而，这么做会破坏模块化的性质。如果其它类假定了被修改的这个类是原来的实现方式的话就会有问题。
  • 重新定义一个更加具体的类，并将代码从原来的类中复制过来再添加新的内容。这么做无法复用代码，原始类发生修改时也无法影响到新的类，并且无法使用 is_a 进行正确的判断。
  • 定义一个类，这个类中的某个成员是原始类的实例。这么做的坏处在于，如果要和原来的类保持相同接口就要写一堆 wrapper 函数，并且也无法使用 is_a 进行判断。但是有时候这么做比直接继承要好。

综上来看继承的好处还是比较明显的。

Method-Lookup Rules, Precisely

这一节讲的是对于 dynamic dispatch 的精确定义，这种定义对大部分 OOP 语言都是适用的。

Dynamic dispatch 的一个描述是，在类 C 中调用 self.m() 可能导致 C 的某个子类中的 m 方法被调用。这可能是 OOP 最独特的性质。

Ruby 中的 self 是一个特殊的关键词，在所有的环境中，它总是指向 当前对象。以下是 dynamic dispatch 的实现，在一个类（这个类不一定和 self.class 相同 ）的方法中：

  • 当要访问 @x 时，在 self 指向的对象中查找 @x。
  • 当要访问 @@x 时，在 self.class 指向的对象中查找 @@x。
  • 当要访问方法时，如果方法的形式为 e0.m(e1, ... , en)：
    1. 对 e0 到 en 求值，假设求值结果为 obj0 到 objn。求值过程中会递归地应用这些规则；
    2. 从 e0 的类中开始找 m，如果没有找到就找它的父类，以此类推，直到找到根类为止。如果还没找到就报错；
    3. 找到 m 后开始执行这个方法，执行前将对应的形参绑定为实参，并且将 self 绑定为 obj0。这一步实现了 dynamic dispatch。因此在 m 中总是会优先使用更加“具体”的方法。

Dynamic dispatch 机制比 closure 要更加复杂，因为要特殊对待 self。但是这不意味着二者孰优孰劣。

其它语言中有 重载（overloading）的概念，但是 Ruby 中没有。这是因为重载需要编译器进行类型分析然后找到最适合调用的函数（没有最优则报错），但是 Ruby 是动态类型的，没有类型分析。

Ruby 中，如果使用同名函数，则总是覆盖掉之前的函数。

Dynamic Dispatch Manually in Racket

这节讲的是在 Racket 中实现 dynamic dispatch。实现上的一些 trick：

  • 在实现类的方法时每个方法额外带一个 self 参数。每次调用的时候将相应的对象作为 self 传进去。在这个方法的实现中可以使用 self。
  • 把类的所有方法（继承的以及自己的）放在同一个列表中，自己的方法放在最前面，根类的方法放在最后面，按照这个顺序排列。这样查找的时候就会正确表现出 dynamic dispatch 的性质。

这样实现 dynamic dispatch 的原理就是，一个对象调用方法的时候是从这个对象的方法列表中找到对应方法的，而又因为越“具体”的方法越先被找到，所以会优先被调用。
