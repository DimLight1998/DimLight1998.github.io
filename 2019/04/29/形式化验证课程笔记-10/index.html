形式化验证课程笔记 10

发表于 2019-04-29 | 更新于 2019-05-06

第十周

Motivation

Given a program composed of many procedures with preconditions and postconditions annotations, how to prove the correctness of the program automatically?

VC Generation: Predicate Transformer

Given an assertion $Q$ and a program $c$, a predicate transformer is a function that produces another assertion.

Given an assertion $Q$ and a program $c$, we can construct a weakest precondition predicate transformer $wp(c, Q)$, such that:

  • $[wp(c, Q)]c[Q]$ is valid
  • for all $P$ such that $[P]c[Q]$ is valid, we have $P\Rightarrow wp(c, Q)$ (hence $wp(c, Q)$ is the weakest).

Similarly we have weakest liberal precondition $wlp(c, Q)$, and it doesn't require termination.

Similarly we have strongest postcondition $sp(c, Q)$.

Imp Language Array Extension

Add array read $x[a]$ and array write $x[a_1]:=a_2$ to the Imp language. Array writing operation will return an updated new array, and the original array was not modified.

While Loop

Review the rule: $\displaystyle{\text{While}\frac{\{P\land b\}c\{P\}}{\{P\}\texttt{while }b\text{ do }c \{P\land \lnot b\}}}$.

The while loops satisfy $\texttt{while }b\texttt{ do }c\equiv\texttt{if }c\texttt{ then }c\texttt{;while }b\texttt{ do }c\texttt{ else skip}$.

We cannot always compute $wlp$ for while loops, and we need loop invariants. Suppose the syntax is $\texttt{while }b\texttt{ do }\{I\}c$, where $\{I\}$ is the loop invariant, often provided by humans. Now we have

However $I$ may not be the weakest.

These are not enough, and we need two additional verification conditions, called $vc(\texttt{while }b\texttt{ do }\{I\}c, Q)$:

  • $\displaystyle{I\land \lnot b\rightarrow Q}$
  • $\displaystyle{I\land b\rightarrow wlp(c, I)}$

Finally, if we want a postcondition holds after a while loop, we need those three formulas to be valid.

Other statements may contains loops, so we need to calculate $vc$ for all kinds of statements:

  • $\displaystyle{vc(\texttt{if }b\texttt{ then }c_2\texttt{ else }c_3, Q) = vc(c_2, Q)\cup vc(c_3, Q)}$
  • $\displaystyle{vc(c_1; c_2, Q) = vc(c_1, wlp(c_2, Q))\cup vc(c_2, Q)}$

Verification Using $wlp$

For an annotated program $\displaystyle{\{P\}c\{Q\}}$, to verify it :

  1. Compute $\displaystyle{P' = wlp(c, Q)}$
  2. Compute $\displaystyle{vc(c, Q)}$
  3. Check $\displaystyle{P\rightarrow P'}$ (this can be done by SMT solver)
  4. Check if all conditions in $\displaystyle{vc(c, Q)}$ is valid

This verification method is sound but not complete, because the precondition of loop may not be the weaknest.

Imp Language Assertion Extension

Assertions are like formal comments. If one program runs to some assertion but doesn't satisfy it, the program will halt.

Imp Language Procedure Extension

We can add procedures into Imp. Preconditions and postconditions can be annotated using requires and ensures like in Dafny.

A program is partially correct if for every procedure in the program, whenever its preconditions hold on entry, its postconditions will be satisfied on exit.

Basic Paths

A basic path is a sequence of instrcutions that begins with a precondition, or a loop invariant, etc. and ends with a postcondition or an assertions, etc.. Basic paths can contain "assume" statements.

A procedure can splitted into several basic paths (with depth-first strategy). If all basic paths are verified, the procedure is verified.

For procedure calls, replace its preconditions with assertions, and introduce fresh variables for its return values, then use assume statements for postconditions.
