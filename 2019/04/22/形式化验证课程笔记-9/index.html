发表于 2019-04-22

第九周

Program Specification

The correctness of programs can be ensured by preconditions and postconditions, which are called program specifications.

Program specifications should be:

  • Declarative
  • Informative
  • Refutable

Notions

  • Environment: valuation of all variables in the program $\sigma$
  • Configuration: $\langle c, \sigma\rangle$, where $c$ is a statement
  • Small-step semantics: $\langle c, \sigma\rangle\rightarrow \langle c', \sigma' \rangle$
  • Big-step semantics: $\displaystyle{\langle c, \sigma \rangle \Downarrow \sigma'}$

Partial Correctness

Hoare Triples

A Hoare triple is in the form of $\displaystyle{\{P\}c\{Q\}}$, where:

  • $c$ is a program
  • $\{P\}$ are the preconditions
  • $\{Q\}$ are the postconditions
  • assertions in $\{P\}$ and $\{Q\}$ are in a first-order theory

Meaning of Hoare Triples

$\displaystyle{\{P\}c \{Q\}}$ is a predicate. If a program starts from an environment satisfying $P$, and if $c$ terminates, then the program will end in an environment satisfying $Q$.

However, we don't know what will happen if the program starts from an environment not satisfying $P$ or $c$ doesn't terminate.

Total Correctness

Total correctness id denoted as $[P]c[Q]$. It is stronger than partial correctness.

$[P]c[Q]$ is a predicate. If a program starts from an environment satisfying $P$, then $c$ terminates, and the program will end in an environment satisfying $Q$.

Semantics of Assertions

Assertions in $P$ or $Q$ are in a first-order theory(for exmaple, $\displaystyle{T_\mathbb Z\cup T_\mathbb A}$). There are three kinds of variables in $P$ of $Q$:

  • variables in $c$
  • variables for quantifiers
  • other logical variables

Because of other logical variables, $P$ or $Q$ cannot be evaluated just by $\sigma$, and we need to introduce assignment $\alpha$.

Semantics of Partial Correctness

$\displaystyle{\sigma\vDash_\alpha \{P\}c \{Q \} }$, if

in which $\sigma\vDash_\alpha X$ means under environment $\sigma$ and assignment $\alpha$, $X$ is true.

Hoare Logic

Rules

The language used in the rules below is the while language.

  • Skip: $\displaystyle{\text{Skip}\frac{}{\{P \} \text{skip}\{ P\} }}$
  • Assignment: $\displaystyle{\text{Asgn}\frac{}{\{Q[a/x]\}x:=a \{Q \}}}$, where $Q[a/x]$ means $Q$ with $a$ substituted for $x$ (replace $x$ with $a$). Using this rule you can get $P$ in $\{P\}c\{Q\}$ given $c$ and $Q$.
  • Strengthening preconditions: $\displaystyle{\text{Pre}\frac{\vdash \{P' \} c \{Q \}\ P\Rightarrow P'}{\{P \} c \{Q \} }}$, if we can prove something, we can still prove it if we have stricter preconditions
  • Weakening postconditions: $\displaystyle{\text{Post}\frac{\vdash \{P \} c \{Q' \} Q'\Rightarrow Q}{\{P \} c \{Q \} }}$, if we can prove something, we can still prove a weaker conclusion with same preconditions
  • Consequence: this is the combination of two rules above $\displaystyle{\text{Conseq}\frac{P\Rightarrow P'\ \vdash \{P' \}c \{Q' \} \ Q'\Rightarrow Q}{\vdash\{P \} c \{Q \} }}$
  • Sequence: $\displaystyle{\text{Seq}\frac{\{P \} c_1 \{P_1 \} \ \{P_1 \} c_2 \{Q \} }{\{P \} c_1;c_2 \{Q \} }}$
  • Conditional: $\displaystyle{\text{If}\frac{\{P\land b \} c_1 \{Q \}\ \{P\land \lnot b \} c_2 \{Q \}}{\{P\} \texttt{if }b\texttt{ then } c_1\texttt{ else }c_2 \{Q \} }}$
  • While loop: $\displaystyle{\text{While}\frac{\{P\land b \} c \{P \} }{\{P \} \texttt{while }b \texttt{ do }c \{P\land \lnot b \} }}$, in which $P$ is the loop invariant; we don't care the termination

Soundness and Completeness of Hoare Logic

Soundness

Hoare logic for partial correctness is sound: if the triple can be derived using the rules, it must be valid.

Completeness

Hoare logic can be completeness (for all valid triples they can be derived). However if the first-order logic is undecidable, Hoare logic is not completeness. So Hoare logic has relative completeness. It is complete for simple languages, but not for complex languages.
