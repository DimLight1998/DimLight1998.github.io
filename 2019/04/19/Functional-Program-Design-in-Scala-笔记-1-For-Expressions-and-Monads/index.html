一些复习

For expression 返回的类型与被迭代的类型上使用 map 函数得到的类型一致。因此 List 上使用 for expression 得到的是 List，而 Map 上使用 for expression 得到的是 Map。

For expression 可以写得比较复杂，可以在花括号中穿插 if，<- 和 =。

Scala 中的 for expression 和 SQL 语言很类似。用得好的话可以在各种数据结构上拥有像 SQL 一样的查询体验。

For Expression 的处理

首先可以注意到 map、filter 以及 flatMap 都可以由 for expression 加以实现。

Scala 编译器则会反过来将代码中的 for expression 翻译回 map、filter 和 flatMap：

  • for (x <- e1) yield e2 会被翻译为 e1.map(x => e2)；
  • for (x <- e1 if f; s) yield e2 会被翻译为 for (x <- e1.withFilter(x => f); s) yield e2，withFilter 是 filter 的 lazy 版本，如果 filter 后面要跟 map 或者 flatMap 操作的话这样做可以加速；
  • for (x <- e1; y <- e2; s) yield e3 会被翻译成 e1.flatMap(x => for (y <- e2; s) yield e3)。

根据上面的规则，可以把任意复杂的 for expression 分解为几个基础高阶函数操作。

另外，只要自己定义的结构具有 map、withFilter 和 flatMap 方法，就可以使用 for expression。

Monad

看下 Scala 下的 monad 的定义是怎样的。

一个 monad 至少需要实现 flatMap，以及能够生成自己类型的 unit。List、Set、Option 和 Generator 都是 monad。unit 并不是某个通用的函数，它只是个概念，对于不同的类型是不一样的：

Monad unit
List List(_)
Set Set(_)
Option Some(_)
Generator single(_)

如果把 monad 看做容器，所谓的 unit 就是将一个元素封装到容器中，使得容器只有这一个元素。

Monad 的 map

map 操作就是将一个 monad 转换成其它类型的 monad！

然而这只是理论上的。因为 Scala 中没有所谓的 unit 函数，因此每个 monad 还是得自己定义对应的 map（本质上是提供对应的 unit 实现）。

Monad 要满足的性质

结合律

m flatMap f flatMap g == m flatMap (x => f(x) flatMap g)

其中 f 和 g 都是把 monad 中包裹的类型转换成另一个 monad 的函数。这个定理表明一个 monad 应该满足结合律。它保证了下面两段代码是等价的：

这样可以让你把嵌套的 for expression 扁平化。

左单位律

unit(x) flatMap f == f(x)

单位的 monad flatMap 上一个函数相当于直接对 monad 中的单位施加这个函数。但是没有应用实例。

右单位律

m flatMap unit == m

flatMap 一个 unit 相当于啥也没做。它保证了 for (x <- m) yield x 和 m 实际上是一样的。

一个数学上正确的 monad 应当满足上面三条定律。不过，即使不满足，在 scala 中依然可以使用 for expression。有些类具有 monad 的特性但是不满足左单位律，却足够实用。

一些结论

可以看到 for expression 实际上被拆分成了 map、flatMap、filter 等操作，这些操作不一定是针对容器的（例如 Generator）。因此 for expression 也可以在不是容器的东西上面使用！
