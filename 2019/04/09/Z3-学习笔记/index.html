Z3 学习笔记

Introduction

Z3 is a theorem prover which can be used to check the satisfiability of logical formulas over theories.

Basic Commands

A Z3 script is a sequence of commands defined by SMT-LIB 2.0 standard.

The syntax is like Racket.

  • echo to print something.
  • declare-const to declare a constant. (No value needed.)
  • declare-fun to declare a function. (No definition needed.)
  • assert to make assertions, which will be used to check satisfiability.
  • check-sat to check the satisfiability of all assertions.
  • get-model to get the assignment if all assertions are satisfiable.

The result of get-model uses define-fun to define functions.

An ite expression stands for an if-then-else branch.

Using Scopes

Z3 uses a global stack to store all declarations. This is useful if you want to prove multiple independent theories using some common declarations.

Use push to enter a new stack frame and use pop to exit the current stack frame.

Configuration

Use set-option to configure Z3. Use reset to remove all configurations and assertions.

Some parameters for set-option:

  • :print-success will toggle printing success for commands.
  • :produce-unsat-cores will toggle generation of unsat cores.
  • :produce-models will toggle model generation.
  • :produce-proofs will toggle proof generation.

Additional Commands

  • display to pretty print an expression.
  • simplify to simplify an expression. Many options are available.
  • define-sort to define a new type. The type can be parameterized.

You may need as as something like type annotations when dealing with parameterized types.

Propositional Logic

Use and, or, xor, not, => (for $\rightarrow$) and = (for $\leftrightarrow$) for logical connectives.

Satisfiability and Validity

If you want to prove $F$ is valid, you can ask Z3 to prove $\lnot F$ is unsatisfiable.

Uninterpreted Functions and Constants

Constants are function taking no parameters, so everything are functions.

Functions in first-order logic should be pure and total.

Use declare-sort to declare a unspecified type.

Arithmetic

Z3 supports integers and real numbers.

Nonlinear Arithmetic

A formula with expression * m n where m and n are neither numbers is called nonlinear. Nonlinear integer arithmetic is undecidable.

Z3 can only solve some of the nonlinear arithmetic problems.

Division

Z3 support real number division /, integer division div, modulo mod and remainder rem.

The result of division by zero is not specified due to all functions need to be total. It can be any result if Z3 wants. You may need a wrapper to avoid this.

Bitvectors

Bitvector theory is a theory for modeling the semantics of unsigned and signed arithmetic in the computer.

The type of bitvector with length $n$ is (_ BitVec n).

Supported operations:

Arithmetic:

  • bvadd addition
  • bvsub subtraction
  • bvneg negation
  • bvmul multiplication
  • bvurem unsigned remainder
  • bvsrem signed remainder
  • bvsmod signed modulo
  • bvshl shift left
  • bvlshr logical shift right
  • bvashr arithmetic shift right

Bitwise:

  • bvor
  • bvand
  • bvnot
  • bvnand
  • bvnor
  • bvxor

Predicate:

u means unsigned while s means signed. le means less than or equal to, others are similar.

  • bvule
  • bvult
  • bvuge
  • bvugt
  • bvsle
  • bvslt
  • bvsge
  • bvsgt

Arrays

Z3 supports array select (array read), array store (array write). It also supports array equality.

Select and Store

Use select for array read and store for array write.

Array write will return a new array with the certain element is replaced.

Constant Arrays

A constant array is an array of which all elements are the same constant.

It can be created as (as const (Array Int Int)) 1.

Array Models

Z3 uses constructor (_ as-array f) to give an interpretation for an array.

Mapping Functions on Arrays

You can do map operation on arrays.

Bags as Arrays

A bag is a data structure like multisets. It keeps the number of each element.

Bags can be implemented in Z3 with arrays.
