Traits

Scala 是单继承的。Scala 中的类只能继承自一个类，但是可以使用 with 继承多个 traits。相当于 Ruby 中的 mixin。

实际上：

If you have multiple classes or traits to inherit, the first one is always extends, and the following >=0 class/trait to be withs.

Java 中的接口和 Scala 中的 trait 类似，区别在于 trait 可以含有自己的状态以及具体的方法。Trait 比类弱的地方在于它本身不能单独被构造，没有构造参数。

特殊的类型

基础类型

Scala 中的几种基础类型：

  • Any 是所有类型的子类型。
  • AnyRef 所有引用类型的子类型。
  • AnyVal 所有值类型的子类型。

和 Java 不同，Java 中的 int 已经是最底层的类型了，但是 Scala 中它还能被 AnyVal 甚至 Any 替代，因此 List(1, true, null) 是合法的（类型为 List[Any]）。

Bottom Type 和 Null Type

Nothing 类型不包含任何值，常用来表示不终止的计算。例如 throw exception 这个表达式的类型就是 Nothing。

Null 类型是所有引用类型的子类型。它具有一个值 null。任何能够出现 AnyRef 的地方都可以出现 Null，但是所有能够出现 AnyVal 的地方都不能出现 Null。

Scala 中有个叫 Unit 的值类型，所谓的 Unit 类型指的是只能取一个值的类型（类似 Null），它无法承载任何信息。Unit 和 Nothing 的区别有些微妙，它们一般用来作为函数的返回类型，区别在于：

  • 如果函数的返回类型是 Unit，表示 逻辑上这个函数不返回有意义的值，可以看作是一个过程。
  • 如果函数的返回类型是 Nothing，表示 这个函数无法终止，唯一退出这个函数的方法是抛出异常。
