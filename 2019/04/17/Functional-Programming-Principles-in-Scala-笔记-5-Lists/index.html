发表于 2019-04-17 | 更新于 2019-04-19

List 上的其它操作

普通操作

  • length，last，init，take，drop 和字面意思一致。
  • xs(n) 可以取出 xs 的第 n 个元素。
  • ++ 连接列表。
  • reverse 翻转列表。
  • 用 updated：更新某个位置上的元素，返回一个新的列表。
  • 用 indexOf 和 contains 寻找元素。

高阶函数操作

除了常见的 map、filter 外：

  • 和 filter 相反的 filterNot。
  • partition 可以根据谓词将列表分成两部分。
  • takeWhile 和 dropWhile。
  • span 结合了 takeWhile 和 dropWhile，但是效率更高，只要一遍遍历。

列表的 reduction 可以通过 reduceLeft 和 reduceRight 实现。这里可能可以用到 Scala 中的一个 语法糖。

Reduce 操作不需要初始累加器，而 fold 操作需要。Fold 比 reduce 泛化抽象程度更高，因为 reduce 的累加器和列表的元素类型要相同。

元组

程序的正确性证明

证明一般是要证明一些性质。和数学归纳法类似，程序的正确性证明要对数据的结构做某种规约。

如果要证明某个操作在列表上会保持某些性质，一般要证明在空列表上保持（Base step），并且在列表 prepend 操作上保持（Inductive step）。
